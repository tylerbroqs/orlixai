# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.5.x (current) | ✓ |
| < 0.5.0 | ✗ |

## Reporting a Vulnerability

**Do not file public GitHub issues for security vulnerabilities.**

To report a security issue, please email:

```
security@orlixai.xyz
```

Include in your report:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested remediation (optional)

You will receive an acknowledgement within **48 hours** and a status update within **7 days**.

We follow a **90-day coordinated disclosure** policy — we will work with you to understand and fix the issue before it is made public.

## Scope

This policy covers:

- The `index.html` single-page application at `orlixai.xyz`
- The Orlix backend API (when applicable)
- Any published npm packages under the `@orlix` scope

Out of scope:

- Issues in third-party dependencies (report those upstream)
- Theoretical vulnerabilities without proof of concept
- Social engineering attacks

## Thank You

We appreciate responsible disclosure and will credit reporters in our release notes (unless you prefer to remain anonymous).
